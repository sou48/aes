　１．3つの評価方式 概要
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　┌────────────────┬─────────────────────────────────────────────────┐
　│ 方式               │ 概要                                                 │
　├────────────────┼─────────────────────────────────────────────────┤
　│ ① KPI方式          │ 期初に数値目標を設定し、達成率で評価する              │
　│                    │ 対象：売上・件数・率など定量指標で測れる業務           │
　├────────────────┼─────────────────────────────────────────────────┤
　│ ② タスクログ方式   │ 四半期中に行ったタスク記録（Redmine）を評価根拠とする │
　│                    │ 対象：プロジェクト型・企画・制作など定性的な業務       │
　├────────────────┼─────────────────────────────────────────────────┤
　│ ③ 行動基準評価方式 │ 行動基準チェックリストへの準拠と上司観察で評価する    │
　│                    │ 対象：定型・反復・チーム完結型の現場オペレーション業務 │
　└────────────────┴─────────────────────────────────────────────────┘

　４．評価方式の選定フローチャート
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　対象者の業務を以下の順で確認し、方式を決定する。

　STEP 1　個人に数値目標を設定できるか？
　　　YES → ① KPI方式
　　　NO  → STEP 2 へ

　STEP 2　四半期ごとに案件・プロジェクト内容が変わるか？
　　　YES → ② タスクログ方式
　　　NO  → STEP 3 へ

　STEP 3　毎日ほぼ同じ定型作業が中心か？
　　　YES → ③ 行動基準評価方式
　　　NO  → 上司・役員と相談の上、最も近い方式を採用

　　（例外）
　　　・役職とともに方式が変わる場合（物流・施工）は
　　　　スタッフ→管理職への昇格時に方式を切り替える
　　　・マーケ・総務のように2方式を軸ごとに分けるケースは
　　　　評価軸（役割遂行・成果創出・組織貢献）ごとに方式を指定する
