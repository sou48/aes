　１．方式の概要とコンセプト
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　個人に数値目標を割り当てにくく、毎日ほぼ同じ定型・反復業務を行う
　現場オペレーション職を対象に、「どのように仕事をしているか」という
　行動・姿勢・習熟度を評価する方式。

　■ 3方式の対比における位置づけ

　┌────────────┬─────────────────────────┬─────────────────────────┐
　│              │ KPI方式との違い               │ タスクログ方式との違い       │
　├────────────┼─────────────────────────┼─────────────────────────┤
　│ 評価の根拠   │ KPIは個人目標だが、           │ タスクログは本人が記録するが、│
　│             │ 行動基準は会社が基準を定める   │ 行動基準は上司が観察する      │
　│ 業務の性質   │ KPIは結果が数値で見えるが、   │ タスクログは多様な案件だが、  │
　│             │ 行動基準は結果よりプロセス重視 │ 行動基準は毎日同じ繰り返し    │
　└────────────┴─────────────────────────┴─────────────────────────┘

　■ 対象部署・対象者

　　・物流事業部　スタッフ層（G1〜G3）　── 倉庫内作業（出荷・入庫・在庫管理）
　　・施工事業部　スタッフ層（G1〜G3）　── 現場施工作業員
　　・総務部　　　スタッフ層（G1〜G3）　── 定型事務担当（勤怠・給与補助・問合せ対応等）
　　・営業部 受注販売課（管理職層・スタッフ層）── 受注処理・出荷手配・ECモール運営支援

　２．評価の構造
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　■ 評価の3軸（全部署共通）

　┌────────────────┬──────────────────────────────────────────────────┐
　│ 評価軸           │ 行動基準評価における意味                               │
　├────────────────┼──────────────────────────────────────────────────┤
　│ ① 役割遂行       │ 定められた手順・ルール・安全基準に沿って正確に           │
　│                  │ 業務を遂行できているか（正確性・手順遵守・安全）         │
　├────────────────┼──────────────────────────────────────────────────┤
　│ ② 成果創出       │ チームの成果指標（誤出荷率・工期遵守など）への貢献と     │
　│                  │ 業務習熟度の向上、自発的な改善提案があるか               │
　├────────────────┼──────────────────────────────────────────────────┤
　│ ③ 組織貢献       │ 後輩サポート・職場環境整備（5S等）・チームへの協力が    │
　│                  │ 自発的にできているか                                    │
　└────────────────┴──────────────────────────────────────────────────┘

　■ 評価の根拠

　　・評価者（部長・係長）の日常的な観察記録が主たる根拠
　　・本人の自己評価は参考情報として活用
　　・チーム全体の数値指標（誤出荷件数等）を文脈として参照
　　・本人の記録は求めないが、面談時に自己振り返りを促す

　４．S/A/B/C 総合判定の考え方
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　■ 各評価軸の集約方法

　　・各チェック項目を S=4点 / A=3点 / B=2点 / C=1点 に換算する
　　・評価軸ごとの項目スコアを単純平均し、軸スコアを算出する
　　・3軸のスコアを部署別ウェイトで加重平均し、総合スコアを算出する

　■ 部署別ウェイト（暫定。部長が調整可）

　┌────────────────┬──────────────┬──────────────┬──────────────┐
　│ 部署             │ ① 役割遂行      │ ② 成果創出      │ ③ 組織貢献      │
　├────────────────┼──────────────┼──────────────┼──────────────┤
　│ 物流スタッフ     │     45%         │     35%         │     20%         │
　│ 施工スタッフ     │     50%         │     30%         │     20%         │
　│ 総務スタッフ     │     40%         │     25%         │     35%         │
　└────────────────┴──────────────┴──────────────┴──────────────┘

　■ 総合評価の判定基準

　┌──────┬───────────────────┐
　│  S   │ 加重平均スコア 3.50 以上 │
　│  A   │ 加重平均スコア 3.00〜3.49│
　│  B   │ 加重平均スコア 2.00〜2.99│
　│  C   │ 加重平均スコア 2.00 未満 │
　└──────┴───────────────────┘

　■ 特例：安全違反・重大ミスがあった場合

　　施工・物流において以下の事態が発生した場合、
　　他の評価に関わらず総合評価を強制的に C に引き下げる：

　　　・安全事故（労災含む）を起こした場合
　　　・重大な誤出荷・施工ミスで顧客に損害を与えた場合
　　　・隠蔽・虚偽報告が判明した場合

　　※ 強制Cの適用はキャリブレーション会議で役員確認を行う

　５．運用手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　STEP 1　評価者による日常観察・記録（四半期通じて継続）

　　・部長・係長は対象者の行動を日常的に観察し、
　　　気づいたことを観察メモ（任意のテキストメモ・Redmineジャーナル等）に残す
　　・特筆すべき良い行動・問題のある行動は具体的な日付と状況を記録する
　　　例：「2026/04/15　Aさんが在庫差異に気づき自発的に報告。原因まで特定した」
　　　例：「2026/05/02　Bさんがピッキングミス。チェックリスト未記入だった」
　　・記録頻度の目安：月2〜3件（良い点・改善点をバランスよく）

　STEP 2　四半期末の自己振り返り（本人）

　　・評価者が本人に「四半期を通じて意識したこと・できたこと・課題」を
　　　面談前に口頭または簡単なメモで伝えるよう促す
　　・本人の記録・ログの提出は必須ではなく、あくまで面談の参考情報

　STEP 3　評価者による行動基準の採点（四半期末）

　　・本設計書の部署別チェックリストに照らして、
　　　各行動基準項目を S/A/B/C で評価する
　　・観察記録がある項目は具体的な事例を根拠として記録する
　　・観察記録がない項目は「A（通常通り問題なし）」を基準とし、
　　　特別な評価変動があった場合のみ上下させる

　STEP 4　面談とフィードバック（四半期末〜翌初）

　　・評価結果と評価根拠を本人に説明する
　　・良かった行動は具体的に言語化して伝える（再現を促す）
　　・改善が必要な行動は指示ではなく「次回こう取り組んでほしい」という
　　　行動目標として提示する
　　・本人の自己振り返りと評価者の見方に大きな差がある場合は
　　　面談内で丁寧にすり合わせを行う

　STEP 5　3軸評価スコアへの変換・通知書への記入

　　・採点結果を3軸ごとに集約し、加重平均スコアで総合評価を算出する
　　・結果を 05_評価フィードバック通知書 に記入し、面談で本人に手渡す

　６．評価者向け　観察のポイントと注意事項
━━━━━━━━━━━━━━━━━━━━━━━━━・・・

　■ 観察時のポイント

　　・「目立つミス」だけを評価根拠にしない
　　　→ 問題なく業務をこなしている = A の行動が積み重なっている価値を認める

　　・直近の印象に引っ張られない（近時効果の回避）
　　　→ 観察メモを四半期通じて残すことで、期末の一時印象に頼らなくてすむ

　　・比較対象を意識する
　　　→「自分の期待値」ではなく「行動基準チェックリストの基準」と照合する

　　・成長の変化を加点要素として活用する
　　　→ 四半期初はBだったが末にはAに改善、という変化はコメント欄に記録し
　　　　次四半期の目標設定に活かす

　■ やってはいけないこと

　　× 観察なしに「なんとなく A」で全項目を埋める（実質評価の放棄）
　　× ミス1件を理由に全評価軸を C に下げる（ハロー効果の誤用）
　　× 本人への事前フィードバックなしに面談で突然 C を告げる
　　× 社歴・年齢・性別などを評価基準に持ち込む

　７．管理職昇格時の方式切替

　翌四半期から評価方式を切り替える。

　│ 部署              │ 切替後の方式       │ 補足                               │
　├──────────────────┼──────────────────┼────────────────────────────┤
　│ 物流事業部スタッフ │ ① KPI方式         │ 在庫差異率・コスト削減等の管理KPIを設定│
　│ 施工事業部スタッフ │ ① KPI方式         │ 工期遵守率・施工件数等の管理KPIを設定  │
　│ 総務部スタッフ    │ ①② 併用           │ 定型業務はKPI、制度整備等はタスクログ  │
　└──────────────────┴──────────────────┴────────────────────────────┘

　　※ 切替タイミング：昇格が決まった翌四半期初の目標設定面談で実施
　　※ 初四半期は KPI / タスクログの記載方法を評価者がサポートする
