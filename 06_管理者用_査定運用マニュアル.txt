  第3章　KPI設定プロセスの管理
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 役員が確認する項目（部長のKPI提出時）

    □ KPIは3つ設定されているか
    □ 各KPIのS/A/B/C達成基準が明記されているか
    □ 測定方法・完了条件が明確か（「頑張る」はNG）
    □ 会社・部署の業績目標に接続しているか
    □ KPI種別の配分が守られているか
      　→ 営業部：成果KPI ×2 ＋ 行動KPI ×1
      　→ 非営業部：行動KPI ×3（改善・再現性中心）

  ■ NG例と修正指示

    ┌───────────────────────┬───────────────────────────────────────────┐
    │ NG例                       │ 修正指示                                     │
    ├───────────────────────┼───────────────────────────────────────────┤
    │「顧客対応を丁寧にする」    │「顧客クレーム0件」「顧客満足アンケート4点以上」等に変更 │
    │ 全部「行動KPI」で数値なし  │（営業）「売上達成率」など成果KPIを2つ設定させる      │
    │ 達成基準がS/A/B/Cで未定義  │ 各段階の基準を四半期初に必ず定義させる              │
    │ 昨期と全く同じKPI          │ 行動KPIの場合は改善・新規取り組みを盛り込むよう指示  │
    └───────────────────────┴───────────────────────────────────────────┘

    一般社員   →  KPI作成  →  部長が承認
    部長（G9） →  KPI作成  →  役員がレビュー・承認

  第4章　四半期評価の実施手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 評価の仕組み（確認用）

    評価は3軸・9〜11項目。各項目をS/A/B/Cで採点。
    加重平均スコアを算出し、総合評価（S/A/B/C）を判定。

    S ＝ 加重平均 3.50以上
    A ＝ 加重平均 3.00〜3.49
    B ＝ 加重平均 2.00〜2.99
    C ＝ 加重平均 2.00未満

    ● 加重平均の計算式
    　加重平均 ＝ Σ（各項目スコア × ウェイト） ÷ 100

    ● スコア変換（S=4 / A=3 / B=2 / C=1）

  ■ 計算例（営業課・10項目）

    項目01：A(3) × 10% ＝ 0.30
    項目02：A(3) × 10% ＝ 0.30
    項目03：B(2) × 10% ＝ 0.20
    項目04：A(3) × 15% ＝ 0.45
    項目05：A(3) × 15% ＝ 0.45
    項目06：B(2) × 10% ＝ 0.20
    項目07：A(3) × 10% ＝ 0.30
    項目08：A(3) ×  7% ＝ 0.21
    項目09：A(3) ×  7% ＝ 0.21
    項目10：S(4) ×  6% ＝ 0.24
    　　　　　　　　　　　────
    　　　　　　　　合計  2.86  → 総合評価：B

  ■ 評価実施のルール

    ・評価は事実・実績に基づくこと（感情・印象で採点しない）
    ・S評価には必ず根拠を書く（「目標の何%達成」等）
    ・C評価には改善計画をセットで準備する
    ・評価後は必ずフィードバック通知書（05_）に転記する

  第5章　キャリブレーション（評価補正）の進め方
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ キャリブレーションとは

    全部署の評価結果を横並びで確認し、部署間・評価者間の不公平を是正する会議。
    役員が主催し、全部長が参加する。評価確定前に必ず実施。

  ■ 会議の進行手順

    STEP 1　事前準備（役員）
      ・全部署の評価一覧を集計する
      　（各社員の：氏名 ／ 部署 ／ Gグレード ／ 加重平均スコア ／ 総合評価）
      ・部署別のS/A/B/C分布を集計する
      ・突出して高い・低い評価者を事前にマークする

    STEP 2　全体共有（15分）
      ・全部署の評価分布を画面・紙で共有
      ・「部署Aは全員A以上」「部署Bは全員B以下」等の偏りを指摘

    STEP 3　個別確認（30〜60分）
      ・S評価の社員：根拠を評価者（部長）に説明させる
      ・C評価の社員：原因と改善計画を確認する
      ・他部署と比較して不自然な評価があれば修正を検討する

    STEP 4　補正の実施
      ・評価修正が必要な場合は、評価者（部長）と合意の上で変更する
      ・修正は「点数の根拠が不十分」「同等実績なのに部署間でS/B差がある」場合に限る
      ・根拠のある評価を一方的に下げてはいけない

    STEP 5　確定・記録
      ・補正後のスコアで評価を確定する
      ・修正した件数・理由を記録に残す

  ■ 部長評価への影響（重要）

    以下の場合、当該部長の評価を減点する：

    ┌──────────────────────────────────┬──────────────────────────────────────┐
    │ 事象                             │ 判断基準                                │
    ├──────────────────────────────────┼──────────────────────────────────────┤
    │ 評価が甘い（インフレ）           │ 部署全員がA以上かつ業績が標準以下の場合  │
    │ キャリブレーション修正が多い     │ 同一部長の評価修正が四半期で3件以上      │
    │ 業績と評価の乖離                 │ 部門目標未達なのにS評価者が複数いる     │
    └──────────────────────────────────┴──────────────────────────────────────┘

  ■ キャリブレーション後の流れ

    確定評価 → 部長が社員に面談（通知書を使用）→ 通知書を2部作成し保管

  第6章　年次総合評価と昇給計算
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 年次総合評価の算出

    年間4回の四半期評価の加重平均スコアを合算して、年次総合評価を決定する。
    評価結果は12月末に確定し、翌年4月支給分より新給与を適用する。
    （12月決算→翌年2月末に決算確定・昇給原資算出→3月にブロック割り振り・人事発令）

    ● 加重方法
      通常年（2027年度〜）：各四半期 25%（均等加重）
      2026年度（初年度）  ：Q2・Q3・Q4 各 33.3%（3四半期均等）

    ※ 加重方法を変更する場合は年度初（1月）に全社員へ告知する。

  ■ 昇給計算の全体フロー

    STEP 1　部署業績レベルの確定（役員が判定）

      ┌────────────────────────────────┬────────────────┐
      │ 判定基準                           │ 調整係数         │
      ├────────────────────────────────┼────────────────┤
      │ 好調：部門目標の120%以上達成       │ ×1.2            │
      │ 標準：部門目標の80〜120%達成       │ ×1.0            │
      │ 不振：部門目標の80%未満            │ ×0.8            │
      └────────────────────────────────┴────────────────┘

    STEP 2　昇給ステップ総本数の確定と部署別配分（役員が決定）

      ◆ 本制度の昇給はすべて「1,500円 × n本（ステップ）」の単位で管理する。
        会社は部署に対して「金額」ではなく「ステップ本数」を配分する。

      ① 全社ステップ予算を決定する（役員）
           全社員数 × 基準ステップ数（A評価相当：4本）＝ 仮総本数
           例）30名 × 4本 ＝ 120本（月額換算：180,000円 / 月）

      ② 部署別の基準本数を算出する
           部署人数（部長含む）× 4本 ＝ 部署基準本数

      ③ 部署業績係数を乗じて配分本数を確定する（端数切り捨て）

      ┌────────────────────────────┬──────┬─────────────────────────────────────────┐
      │ 部署                        │ 人数  │ 基準本数 × 係数 ＝ 配分本数（例）          │
      ├────────────────────────────┼──────┼─────────────────────────────────────────┤
      │ 営業部（営業課）            │  4名  │  4×4 × 1.0 ＝ 16本                        │
      │ 営業部（受注販売課）        │  2名  │  2×4 × 1.0 ＝  8本                        │
      │ クリエイティブ事業部        │  4名  │  4×4 × 1.0 ＝ 16本                        │
      │ マーケティング企画部        │  3名  │  3×4 × 1.2 ＝ 14本（好調・端数切捨て）     │
      │ 施工事業部                  │  3名  │  3×4 × 0.8 ＝  9本（不振・端数切捨て）     │
      │ 物流事業部                  │  3名  │  3×4 × 1.0 ＝ 12本                        │
      │ 事業推進部                  │  2名  │  2×4 × 1.0 ＝  8本                        │
      │ 総務部                      │  2名  │  2×4 × 1.0 ＝  8本                        │
      └────────────────────────────┴──────┴─────────────────────────────────────────┘

      ※ 上記は例示。実際の配分は役員が毎年度初に確定し、各部長へ書面で通知する。
      ※ 配分本数は「上限」。使い切れなかった本数は翌年度に繰り越さない。
      ※ 部長（G9）自身の昇給ステップは役員が直接決定し、部署配分本数に含める。

    STEP 3　部長による部下へのステップ配分（部長が実施）

      部長は配分された総本数を、個人評価ランクの範囲内で部下に割り振る。

      ● 評価ランク別の配分目安（1人あたり）と月額換算

      ┌──────┬──────────────────────────┬──────────────────────────┐
      │ 評価  │ 配分ステップ目安（1人）        │ 月額増加（目安）              │
      ├──────┼──────────────────────────┼──────────────────────────┤
      │  S   │ ６本（最大8本まで裁量可）       │ ＋9,000円〜12,000円 / 月      │
      │  A   │ ４本（最大5本まで裁量可）       │ ＋6,000円〜7,500円 / 月       │
      │  B   │ ２本（最大3本まで裁量可）       │ ＋3,000円〜4,500円 / 月       │
      │  C   │ ０本（現状維持）               │ 変動なし                     │
      └──────┴──────────────────────────┴──────────────────────────┘

      ● 配分ルール
        ・配分本数の合計が「部署への割当本数」を超えてはいけない
        ・評価ランクをまたぐ配分は禁止（B評価者にA相当の本数は不可）
        ・ランク内の裁量は ±1〜2本。根拠を役員に説明できること
        ・余った本数は会社へ返却（個人への追加配分に使わない）

    STEP 4　部長裁量案を役員へ提出・承認（役員が確認）

      ・部長は「誰に何本配分したか」の一覧を役員に提出する
      ・役員は以下を確認して承認する：
        □ 配分合計が割当本数以内か
        □ 評価ランクと配分本数に矛盾がないか
        □ 裁量加算に根拠があるか（口頭説明または備考記入）

    STEP 5　新等級（数値）の確定と月給計算

      新しい数値等級 ＝ 現在の数値等級 ＋ 最終ステップ数
      新月給 ＝ 部署基準額 ＋（新しい数値等級 − 1）× 1,500円 ＋ 役職手当

      ● 部署基準額（等級1の月給）
        営業課：250,000円　　受注販売課：190,000円　　クリエイティブ：300,000円
        マーケティング：230,000円　　物流：230,000円　　施工：230,000円　　総務：230,000円

      ● 役職手当
        副主任（G4）：＋5,000円　　主任（G5）：＋10,000円　　係長（G6）：＋20,000円
        課長（G7）：＋30,000円　　次長（G8）：＋40,000円　　部長（G9）：＋50,000円　　本部長（G10）：＋100,000円

  ■ 計算例

    【前提】物流事業部 3名・部署業績標準（×1.0）
      部署への配分本数：3名 × 4本 × 1.0 ＝ 12本

    【各人への配分】
      山口（部長・G9）　評価A → 4本　（役員が直接決定・配分12本に含む）
      中村 優（G2）　　 評価S → 6本
      西本 隆（G1）　　 評価B → 2本
      　　　　　　　　　　　　　　　合計 12本 ＝ 割当12本 → 過不足なし ✓

    【中村 優の月給計算】（G2・数値等級22・物流事業部）
      ① 配分ステップ：6本（S評価）
      ② 新数値等級：22 ＋ 6 ＝ 28
      ③ 新月給：230,000 ＋（28−1）×1,500 ＋ 職能手当40,000＋（28−1）×1,500
               ＝ 230,000 ＋ 40,500 ＋ 40,000 ＋ 40,500 ＝ 351,000円
               ※ 月額増加 ＝ 6本 × 1,500円 × 2（基本給＋職能手当）＝ ＋18,000円 / 月

  第7章　昇格判定の手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 昇格の2条件（両方満たすことが必須）

    ① 役割充足判定　：次のGグレードの役割定義を満たしているか
    ② ポスト判定　　：組織上の空きポストがあるか

  ■ 昇格フロー

    STEP 1　部長から昇格候補の提案を受ける（随時）
    STEP 2　01_等級定義を参照し、次グレードの要件を満たすか確認する
    STEP 3　ポストの空き状況を確認する
    STEP 4　承認権限に応じた手続きを行う（下表）

    ┌──────────────┬──────────────────────────────────────────────┐
    │ 昇格先グレード   │ 承認手続き                                             │
    ├──────────────┼──────────────────────────────────────────────┤
    │ G1 → G2 → G3  │ 部長判断のみ（役員への報告は任意）                       │
    │ G3 → G4 〜 G6 │ 役員レビュー → 承認                                      │
    │ G6 → G7 〜 G8 │ 役員承認（昇格理由・根拠を書面で確認）                    │
    │ G8 → G9       │ 役員承認 ＋ 代表取締役の最終確認                          │
    │ G9 → G10      │ 代表取締役による任命（数値等級・評価基準なし・純粋なポスト任命）│
    └──────────────┴──────────────────────────────────────────────┘

  ■ 昇格最低条件（等級定義より抜粋）

    G1→G2：G1在籍2四半期以上 ／ 直近2四半期 A以上 ／ 部長推薦
    G2→G3：G2在籍4四半期以上 ／ 直近4四半期 A以上（うち1四半期 S以上）
    G3→G4：G3在籍2四半期以上 ／ 直近2四半期 S以上 ／ ポスト空き確認 ／ 役員レビュー
    G4→G5：G4在籍4四半期以上 ／ 直近4四半期 A以上（うち2四半期 S以上）
    G5→G6：G5在籍4四半期以上 ／ 直近4四半期 A以上 ／ 係長候補能力確認
    G6→G7：G6在籍4四半期以上 ／ 直近4四半期 A以上（うち2四半期 S以上）
    G7→G8：G7在籍4四半期以上 ／ 直近4四半期 S以上 ／ 役員承認
    G8→G9：G8在籍4四半期以上 ／ 直近4四半期 S以上 ／ 代表承認

  ■ ポストがない場合の対応（昇格保留時）

    ・数値等級の上限を引き上げる（拡張ゾーン適用）
    ・役割給・特別手当で処遇する
    ・昇格保留の理由を本人に明示する（「評価が足りない」ではなく「ポストがない」）

  ■ 昇格保留時の本人への説明例

    「現時点でのあなたの評価・能力は次グレードの要件を満たしています。
     ただし、現在対応するポストに空きがないため、昇格は保留とします。
     今後ポストが生じた際には、最優先で対応します。
     処遇については、数値等級の上限を引き上げる形で対応します。」

  第8章　部長評価の実施方法
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 部長（G9）の評価は役員が実施する

    部長評価は通常社員と異なる専用項目で行う。
    ウェイト：役割遂行 ×1.5 ／ 成果創出 ×1.0 ／ 組織貢献 ×1.2

  ■ 評価項目（7項目）

    ┌────┬─────────────────────────────────┬──────┐
    │ No │ 評価項目                                 │ ウェイト│
    ├────┼─────────────────────────────────┼──────┤
    │ 01 │ 部門方針・計画の策定・実行               │ ×1.5  │
    │ 02 │ マネジメントの質（評価・育成）           │ ×1.5  │
    │ 03 │ 経営視点の意思決定・課題提言             │ ×1.5  │
    │ 04 │ 部門KPI達成率                            │ ×1.0  │
    │ 05 │ コスト管理・利益貢献                     │ ×1.0  │
    │ 06 │ 部門文化・チームエンゲージメント         │ ×1.2  │
    │ 07 │ 他部署・役員との協働                     │ ×1.2  │
    └────┴─────────────────────────────────┴──────┘

  ■ 加重平均の計算（部長用）

    ウェイトの合計 ＝ 1.5×3 ＋ 1.0×2 ＋ 1.2×2 ＝ 4.5 ＋ 2.0 ＋ 2.4 ＝ 8.9

    加重平均 ＝ Σ（スコア × ウェイト） ÷ 8.9

    例）全項目 A（3点）の場合：3 × 8.9 ÷ 8.9 ＝ 3.00 → 総合評価 A

  ■ 部長評価での注意点

    ・02「マネジメントの質」は、キャリブレーション結果を反映して採点する
      → 修正件数が多い部長は B 以下を検討すること
    ・部長自身が提出した自己評価とのギャップが大きい場合、その理由を面談で説明する
    ・部長評価の面談は役員が直接実施する（通知書フォーマットは同じものを使用）

  第9章　NG行為・よくある間違い
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 絶対にやってはいけないこと

    ✕ 評価ランクを逸脱した昇給配分
      → B評価者にA評価相当の昇給を与えることは制度違反

    ✕ キャリブレーションなしで評価を確定する
      → 部長一人の判断を最終とすることはできない

    ✕ 昇格条件を満たしていない社員のポスト外昇格
      → 「好きだから」「長く働いているから」は理由にならない

    ✕ C評価の社員に改善計画なしで通知する
      → C評価は必ず具体的なアクションをセットで提示する

    ✕ 部長が自部署社員全員をS・Aにする
      → S評価には必ず根拠が必要。キャリブレーションで是正される

  ■ よくある間違い

    △ 評価スケールの混同
      → 「A = 100点」ではない。A = 期待通り（3点）が基本ライン
      → S は「明確な超過実績があるとき限定」で使う

    △ 部署業績調整の見落とし
      → 個人評価を決めた後、部署業績係数（×0.8〜1.2）の適用を忘れない

    △ 数値等級の上限到達 ＝ 昇格 と誤解
      → 上限到達は昇格ではなく「拡張ゾーン適用」で対応する

    △ 面談を省略する
      → 面談は制度上の義務。通知書への署名がないものは無効扱いとする
