　本書は、査定評価制度を運用するにあたり、役員および部長が実施すべき
　手順、判断基準、注意事項をまとめた運用マニュアルです。

　現時点では、固定的な等級グレード制度は採用しません。
　給与改定は「年次総合評価に基づく昇給ブロック配分」で決定します。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 制度の目的（優先順位）

    ① 価値観・行動基準の統一
    ② 公平な昇給決定
    ③ 人材育成
    ④ マネジメント責任の明確化
    ⑤ 業績向上

  ■ 管理者が担う役割

    ┌─────────────────────────────────────────────────────────────┐
    │ 役割                           │ タイミング             │ 詳細            │
    ├─────────────────────────────────────────────────────────────┤
    │ ① 評価運用ルールの遵守確認     │ 毎期                  │ → 第3〜5章      │
    │ ② 部長KPIのレビュー・承認      │ 四半期初              │ → 第3章         │
    │ ③ 評価分布・根拠文の確認       │ 四半期末              │ → 第5章         │
    │ ④ 昇給原資の部署別配分決定     │ 翌年3月               │ → 第6章         │
    │ ⑤ 部署内配分の妥当性確認       │ 翌年3月               │ → 第6章         │
    │ ⑥ 部長評価の実施               │ 毎期                  │ → 第8章         │
    │ ⑦ 昇格・役職変更の判断         │ 随時                  │ → 第7章         │
    └─────────────────────────────────────────────────────────────┘

  ■ 重要な前提

    ・評価は四半期ごとに実施する
    ・昇給反映は年1回、翌年4月とする
    ・固定的な等級グレード制度は現時点では採用しない
    ・役職の呼称と責任範囲は維持する

  第2章　年間・四半期カレンダー
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 通常年の流れ

    1月〜3月   Q1運用
    4月〜6月   Q2運用
    7月〜9月   Q3運用
    10月〜12月 Q4運用

    12月末     年次総合評価確定
    翌年2月末  決算確定・昇給原資確認
    翌年3月    役員が部署別に昇給ブロックを配分
               部長が部署内で各社員へ配分
    翌年4月    新給与適用

  ■ 2026年度の特例

    ・制度開始は2026年4月
    ・Q1（1〜3月）は制度導入準備期間としてスキップ
    ・Q2、Q3、Q4の3四半期で年次総合評価を算出
    ・初回昇給反映は2027年4月

  ■ 四半期ごとの運用手順

    WEEK 1〜2   KPI設定または目標確認
    WEEK 3〜12  実務期間
    WEEK 13     評価入力
    WEEK 14     キャリブレーション
    WEEK 15     面談実施
    WEEK 16     通知書保管

  第3章　評価方式ごとの管理ポイント
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ KPI方式

    対象例：営業課、受注販売課、施工事業部管理職、物流事業部管理職、全部署の部長職

    管理ポイント：
    ・目標が測定可能であること
    ・S/A/B/C基準が事前に明記されていること
    ・会社方針や部署目標と接続していること

  ■ タスクログ方式

    対象例：クリエイティブ事業部、事業推進部

    管理ポイント：
    ・Redmine記録が継続されていること
    ・タスク種別、難易度、成果が記録されていること
    ・単なる件数評価ではなく、難易度と成果内容を見て判断すること

  ■ KPI＋タスクログ併用

    対象例：マーケティング企画部、総務部管理職

    管理ポイント：
    ・定量目標と定性実績の両方を見ること
    ・どちらか一方に偏らないこと

  ■ 行動基準評価方式

    対象例：施工事業部スタッフ、物流事業部スタッフ、総務部スタッフ

    管理ポイント：
    ・日常観察の事実で評価すること
    ・チェックリスト項目に沿って判断すること
    ・強制C条件に該当する場合は例外なく反映すること


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  第4章　四半期評価の実施手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 基本ルール

    ・評価は事実・実績ベースで行う
    ・感情、印象、好き嫌いで採点しない
    ・S評価とC評価には必ず根拠文を付ける
    ・C評価には改善方針をセットで示す
    ・評価項目は部署固定ではなく、部長が四半期初に本人の担当業務に応じて選択する

  ■ 評価軸

    ① 役割遂行
    ② 成果創出
    ③ 組織貢献

  ■ 評価項目の選択ルール

    ・部長は `30_評価項目_部署別設計.txt` の候補項目から各社員の適用項目を選ぶ
    ・原則として3軸すべてから項目を選定する
    ・実際に担当していない業務の項目は選ばない
    ・選定した項目は四半期初に確定し、担当変更がない限り期中変更しない
    ・選定理由は本人に説明できる状態にしておく

  ■ 評価スケール

    S ＝ 期待を大きく上回る
    A ＝ 期待通り
    B ＝ やや不足
    C ＝ 不足・改善が必要

  ■ 面談時の必須説明

    ・評価結果
    ・その評価にした根拠
    ・本人の強み
    ・改善点
    ・次四半期のアクション

  第5章　キャリブレーションの進め方
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 方針

    従来の「役員＋全部長の横断会議」は行わない。
    各部長の自己チェックと、役員による個別レビューで運用する。

  ■ STEP 1　部長による評価分布チェック

    各部長は評価確定前に、自部署のS/A/B/C比率を確認する。

    ┌──────┬──────────────┐
    │ 評価 │ 推奨比率       │
    ├──────┼──────────────┤
    │ S    │ 10〜15%        │
    │ A    │ 30〜40%        │
    │ B    │ 40〜50%        │
    │ C    │ 5〜10%         │
    └──────┴──────────────┘

    ※ 対象者3名以下の部署は比率ルール免除

  ■ STEP 2　S/C評価の根拠文提出

    S評価者とC評価者について、3〜5行の根拠文を提出する。

    記載項目：
    ・氏名
    ・部署
    ・評価
    ・具体的な行動、成果、事実
    ・今後への期待または改善方針

  ■ STEP 3　役員による個別レビュー

    以下のいずれかに該当する場合のみ、役員が当該部長と個別レビューする。

    ・S比率超過
    ・全員A以上
    ・業績と評価の乖離
    ・根拠文不十分
    ・C評価過多

    他部署の部長は参加しない。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  第6章　年次総合評価と昇給ブロック配分
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 基本方針

    昇給は、固定の等級テーブルではなく、昇給ブロック配分で決定する。

  ■ 手順

    STEP 1　Q1〜Q4の四半期評価を集計する
    STEP 2　年次総合評価を確定する
    STEP 3　決算確定後、役員が全社の昇給原資を確認する
    STEP 4　役員が各部署へ昇給ブロックを配分する
    STEP 5　部長が部署内で各社員へ配分する
    STEP 6　役員が配分結果の妥当性を確認する
    STEP 7　翌年4月給与へ反映する

  ■ 評価ごとの配分目安

    ┌──────┬──────────────────────┐
    │ 評価 │ 昇給ブロック配分目安     │
    ├──────┼──────────────────────┤
    │ S    │ 6〜8本相当               │
    │ A    │ 4〜5本相当               │
    │ B    │ 2〜3本相当               │
    │ C    │ 0〜1本相当               │
    └──────┴──────────────────────┘

  ■ 配分時の注意

    ・同評価内の調整は可
    ・評価ランクを超えた逸脱配分は不可
    ・部署業績係数を考慮する
    ・部長自身の配分は役員が決定する

  第7章　昇格・役職変更の扱い
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 現時点の位置づけ

    役職は維持するが、役職と連動した固定等級制度は採用しない。

  ■ 昇格・役職変更の判断軸

    ・現職で求められる役割を安定して果たしているか
    ・上位役職の役割を実質的に担えているか
    ・組織上のポスト必要性があるか
    ・四半期評価の蓄積結果に問題がないか

  ■ 現時点の扱い

    ・昇格判定の詳細手順は未確定
    ・運用上は部長提案、役員確認、必要に応じて代表承認で扱う
    ・詳細制度は別途設計対象とする

  ■ 重要注意

    昇給配分と昇格は別概念である。
    高評価であっても、ただちに昇格決定とはしない。

  第8章　部長評価の実施方法
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 部長評価の原則

    全部署の部長は、役員が評価する。
    評価方式は「KPI方式＋部署独自評価3項目」を用いる。
    ウェイトは 役割遂行 45% ／ 成果創出 30% ／ 組織貢献 25% を基本とする。

  ■ 見るポイント

    ・部門目標の達成状況
    ・評価運用の公正さ
    ・人材育成と組織運営
    ・他部署や役員との連携
    ・新しいサービス・商品・仕組みを生み出そうとしているか
    ・メンバーから「こんなことをやってみたい」を引き出せているか

  ■ 部長評価で減点対象となる例

    ・部署全員A以上かつ業績が標準以下
    ・根拠文不備が四半期で2件以上
    ・役員レビューによる修正が四半期で3件以上
    ・部門目標未達なのにS評価者が複数いる

  ■ 部長評価で加点対象となる例

    ・既存業務の延長ではない新サービス・新商品・新仕組みの種を生み出している
    ・将来の売上につながるテーマを部門から創出している
    ・部下からの提案や挑戦案を継続的に引き出している
    ・挑戦が個人依存でなく、部門運営の仕組みに落ちている

  第9章　NG行為・よくある誤り
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ NG行為

    ・評価ランクを超えた昇給配分
    ・キャリブレーション前の独断確定
    ・根拠のないS評価
    ・改善方針のないC評価
    ・役職変更と昇給配分を混同すること

  ■ よくある誤り

    ・Aを高評価と捉えすぎる
      → Aは標準ライン

    ・部長同士で横断会議を開く前提で進める
      → 現行方式は役員個別レビュー

    ・昇給を固定テーブルに当てはめて決めようとする
      → 現行方式は昇給ブロック配分

    ・役職があるので等級制度も確定していると誤解する
      → 等級グレード設計はペンディング
