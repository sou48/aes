# AGENTS.md — 査定評価システム ファイル構成ガイド

タイガーラック株式会社 給与査定評価システムの設計ドキュメント一覧です。
AIエージェントや担当者がファイルの役割・参照関係を把握するために使用します。

---

## ⚠️ 作業ルール（必読）

### OVERVIEW.md・運用資料の連動更新ルール

**制度設計に変更が生じた場合、必ず `OVERVIEW.md` に加えて `50_管理者用_運用マニュアル.txt` と `60_社員向け_制度案内資料.txt` も必要に応じて連動更新すること。**

`OVERVIEW.md` は他のAIエージェントや担当者が設計を引き継ぐための唯一の概要書であり、
`50_管理者用_運用マニュアル.txt` は管理者向けの実運用基準、
`60_社員向け_制度案内資料.txt` は社員向け説明の基準資料である。
個別の設計ファイルだけを更新してこれらを放置すると、制度説明と運用手順に乖離が生じる。

#### 更新が必要なタイミング

- 評価方式・ルールの追加・変更・廃止
- グレード・等級の考え方、またはその保留方針の変更
- 昇給ブロック配分ルール・ステップ単価の変更
- 部署・担当者・ポスト構成の変更
- キャリブレーションや昇給フローの手順変更
- 新規ファイルの追加（ファイル構成クイックリファレンスを更新）
- 未設計領域が解決された場合（セクション13「未設計・未確定の領域」から削除）
- 新たな設計上の意思決定が行われた場合（セクション12「意思決定ログ」に追記）

#### 更新手順

1. 個別の設計ファイル（21_〜・23_〜 など）を変更する
2. `OVERVIEW.md` の該当セクションを変更内容に合わせて更新する
3. `50_管理者用_運用マニュアル.txt` の運用手順・説明を更新する
4. `60_社員向け_制度案内資料.txt` の社員向け説明を更新する
5. `OVERVIEW.md` 末尾の「最終更新」日付を更新する

> `AGENTS.md`（ファイル構成・命名ルール）、`OVERVIEW.md`（制度概要）、`50_管理者用_運用マニュアル.txt`（管理者運用）、`60_社員向け_制度案内資料.txt`（社員説明）は常に最新状態を保つこと。

## システム概要

- **評価サイクル**：四半期ごと（Q1〜Q4）、年4回
- **昇給タイミング**：4月（12月決算→2月末決算確定→3月ブロック割り振り→4月給与反映）
- **等級グレード設計**：現時点ではペンディング（役職は維持し、昇給はブロック配分で決定）
- **評価方式**：KPI方式 / タスクログ方式 / 行動基準評価方式（部署・役職によりいずれかを適用）
- **評価3軸**：役割遂行・成果創出・組織貢献（部署別ウェイト）

## ファイル命名・ナンバリングルール

### 番号帯とカテゴリの対応

| 番号帯 | カテゴリ | 格納場所 |
|---|---|---|
| 00〜03 | 基盤情報（会社・部署・役職の前提情報） | `基本情報/` フォルダ内 |
| 10〜19 | 等級グレード方針・関連設計 | ルート直下 |
| 20〜29 | 評価方式 設計ドキュメント | ルート直下 |
| 30〜39 | 評価項目 設計ドキュメント | ルート直下 |
| 40〜49 | テンプレート・記入シート | ルート直下 |
| 50〜59 | 運用マニュアル・ガイド | ルート直下 |
| 60〜69 | 社員向け資料 | ルート直下 |

### ファイル名のフォーマット

```
{番号}_{カテゴリキーワード}_{内容説明}.{拡張子}
```

- 番号は2桁固定（01、20 など）
- カテゴリキーワードは名詞で統一（「管理者用」「評価方式」「フィードバック通知書」など）
- 内容説明は省略可。同カテゴリに複数ファイルある場合に付与（「テンプレート」「記入例_部署名」など）
- 拡張子：設計・マニュアル類は `.txt`、記入シート類は `.csv`

### 同番号帯内での採番ルール

- 同カテゴリ内で「概要・全体像」→「個別詳細」の順で採番する
  - 例）20: マッピング（全体）→ 21: KPI（個別）→ 22: タスクログ（個別）→ 23: 行動基準（個別）
- テンプレート本体（40）→ 通知書テンプレート（41）→ 記入例（42）→ 各方式シート（43〜44）→ 計算シート（45）
- 番号は10刻みで予約し、カテゴリ間に空き番号を設けない（追加時は既存番号帯の空き番号を使う）

### 新規ファイル追加時のルール

1. カテゴリを上記対応表で確認し、該当番号帯を決める
2. 同番号帯内の既存ファイルと照合し、連番で採番する
3. `AGENTS.md` のファイル一覧・参照フローを更新する

## 更新履歴メモ

- 昇給タイミング：1月 → **4月** に修正済み（12月決算→2月末確定→3月割り振り→4月反映）
- 等級グレード設計：現時点では **ペンディング** に変更済み（昇給はブロック配分を基準とする）
- 評価方式：KPI・タスクログの2方式 → **行動基準評価方式を追加し3方式**に拡張済み
- 昇給ステップ数：S=6〜8本・A=4〜5本・B=2〜3本・C=0〜1本
- キャリブレーション：全部長会議方式 → **評価分布チェック＋役員個別レビュー方式**に変更済み
