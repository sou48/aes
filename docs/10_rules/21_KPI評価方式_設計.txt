━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
　１．方式の概要とコンセプト
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　四半期初に「何をどこまで達成するか」を本人と上司が合意し、
　期末に達成率・完了状態で評価する方式。

　■ 対象部署・対象者

　　・営業部　営業課（全グレード）
　　・営業部　受注販売課（全グレード）
　　・施工事業部　管理職層（G4以上）
　　・物流事業部　管理職層
　　・マーケティング企画部（成果創出軸のみ）　※タスクログ方式と併用
　　・総務部　管理職層　　　　　　　　　　　 ※タスクログ方式と併用
　　・全部署の部長職

　２．KPIの種別定義
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　■ 3種別の定義

　┌────────────────┬──────────────────────────────────────────────────┐
　│ 種別               │ 内容・特徴                                             │
　├────────────────┼──────────────────────────────────────────────────┤
　│ 成果KPI（数値）    │ 売上・粗利・受注件数・ROAS など定量的な結果指標        │
　│                    │ 達成率（%）で評価できる。数値が上ぶれ・下ぶれして評価  │
　├────────────────┼──────────────────────────────────────────────────┤
　│ 成果物KPI          │ 制作物・レポート・マニュアルなど「完成物」が評価根拠   │
　│（アウトプット）    │ 完了条件（いつまでに・何が揃えば完了か）を明記する     │
　├────────────────┼──────────────────────────────────────────────────┤
　│ 行動KPI            │ プロセス改善・再現性・習慣化など行動・取り組みを評価   │
　│（改善・再現性）    │ 完了条件と成果との接続（なぜ重要か）を明記する         │
　└────────────────┴──────────────────────────────────────────────────┘

　■ 部署別 KPI構成ルール

　┌────────────────────┬──────────────────────────────────────────────┐
　│ 部署               │ KPI構成（四半期3本）                               │
　├────────────────────┼──────────────────────────────────────────────┤
　│ 営業課             │ 成果KPI × 2　＋　行動KPI × 1                     │
　│ 施工事業部（管理職）│ 行動KPI × 3（安全・品質・工期中心）              │
　│ 物流事業部（管理職）│ 行動KPI × 3（正確性・効率性中心）               │
　│ マーケティング企画部│ 成果KPI × 2　＋　行動KPI × 1                     │
　│ 総務部（管理職）   │ 行動KPI × 3（正確性・コンプライアンス中心）       │
　│ 部長（全部署）     │ 成果KPI × 1（部門KPI）＋ 行動KPI × 2            │
　└────────────────────┴──────────────────────────────────────────────┘


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
　３．KPI設定の手順とルール
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　STEP 1　本人がKPIを作成する（四半期初 第1週）

　　・43_KPIシート_テンプレート.csv を使用する
　　・各KPIに以下を必ず記入する：
　　　　─ KPI種別（成果 / 成果物 / 行動）
　　　　─ KPIタイトル（一言で内容が伝わる表現）
　　　　─ KPIの定義・詳細内容（背景・目的・範囲）
　　　　─ 完了条件（「何が揃えば完了か」を箇条書きで明記）
　　　　─ S / A / B / C 各基準（数値または状態を明記）

　STEP 2　部長による承認・調整（四半期初 第1〜2週）

　　・各KPIが以下を満たしているか確認する
　　　　─ 完了条件が明確か（曖昧な表現を避ける）
　　　　─ S/A/B/C の境界が明確か
　　　　─ 成果との接続が説明できるか
　　　　─ 本人が主体的に取り組める内容か（外部要因依存が高すぎない）
　　・修正が必要な場合は本人にフィードバックし再提出させる
　　・承認後は変更禁止（変更する場合は「KPI変更申請」として理由を記録する）

　STEP 3　中間確認面談（四半期中盤・第6〜7週）

　　・KPI進捗を確認し、達成見込みを本人と共有する
　　・外部環境の変化（大型案件の突発など）でKPI変更が必要な場合のみ修正可
　　　→ 変更記録欄に変更理由・変更日・変更前後の基準を記録する

　STEP 4　期末の実績記入と自己評価（四半期末 最終週）

　　・本人が実績値・自己評価（S/A/B/C）を記入する
　　・根拠（数値・完了エビデンス・URL等）を必ず記載する

　STEP 5　部長によるKPI評価（四半期末〜翌週初）

　　・実績値と完了条件を照合し、S/A/B/C を判定する
　　・自己評価と評価者評価に差がある場合は面談で根拠を説明する
　　・評価コメント欄に判定理由を記入する


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
　４．S/A/B/C 判定基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　■ 成果KPI（数値）の判定

　┌──────┬──────────────────────────────────────────────────────────┐
　│  S   │ 目標値の 120% 以上を達成                                       │
　├──────┼──────────────────────────────────────────────────────────┤
　│  A   │ 目標値の 100〜119% を達成                                      │
　├──────┼──────────────────────────────────────────────────────────┤
　│  B   │ 目標値の 80〜99% を達成                                        │
　├──────┼──────────────────────────────────────────────────────────┤
　│  C   │ 目標値の 80% 未満                                              │
　└──────┴──────────────────────────────────────────────────────────┘

　　※ 外部要因（市場環境・大口顧客の事情）で目標未達の場合、
　　　プロセス評価（訪問件数・提案活動）を補足情報として加味できる。
　　　ただし最終判定は評価者が行い、根拠をコメント欄に記録する。

　■ 成果物KPI（アウトプット）の判定

　┌──────┬──────────────────────────────────────────────────────────┐
　│  S   │ 完了条件を全て満たした上で、期日前に完了。                     │
　│      │ かつ想定を超える品質・波及効果があった                         │
　├──────┼──────────────────────────────────────────────────────────┤
　│  A   │ 完了条件を全て満たし、期日通りに完了                           │
　├──────┼──────────────────────────────────────────────────────────┤
　│  B   │ 完了条件の一部未達、または期日から1週間以内の遅延で完了        │
　├──────┼──────────────────────────────────────────────────────────┤
　│  C   │ 完了条件の主要部分が未達、または期日を1週間以上超過・未完了    │
　└──────┴──────────────────────────────────────────────────────────┘

　■ 行動KPI（改善・再現性）の判定

　┌──────┬──────────────────────────────────────────────────────────┐
　│  S   │ 完了条件を全て満たし、さらに他者・他部署への再現・展開がある   │
　├──────┼──────────────────────────────────────────────────────────┤
　│  A   │ 完了条件を全て満たして完了                                     │
　├──────┼──────────────────────────────────────────────────────────┤
　│  B   │ 完了条件の一部が未達（着手済みだが完了まで至らなかった）       │
　├──────┼──────────────────────────────────────────────────────────┤
　│  C   │ ほぼ未着手、または完了条件の大部分が未達                       │
　└──────┴──────────────────────────────────────────────────────────┘


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
　５．3軸評価スコアへの変換
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　KPIの評価結果（3本）を3軸評価（役割遂行・成果創出・組織貢献）に変換する。

　■ 変換マッピング（部署・KPI種別に応じて設定）

　┌────────────────────┬──────────────────────────────────────────┐
　│ 3軸評価              │ KPIとの対応                                  │
　├────────────────────┼──────────────────────────────────────────┤
　│ ① 役割遂行           │ 行動KPI（プロセス・精度・計画実行）の評価結果 │
　├────────────────────┼──────────────────────────────────────────┤
　│ ② 成果創出           │ 成果KPI / 成果物KPI の評価結果               │
　├────────────────────┼──────────────────────────────────────────┤
　│ ③ 組織貢献           │ 行動KPI（共有・育成・横断対応）の評価結果    │
　└────────────────────┴──────────────────────────────────────────┘

　■ スコア計算（S=4点 / A=3点 / B=2点 / C=1点）

　　加重平均スコア ＝ Σ（各KPIスコア × ウェイト） ÷ Σウェイト

　　部署別ウェイトは 30_評価項目_部署別設計.txt に準拠する。

　■ 総合評価の判定

　┌──────┬───────────────────┐
　│  S   │ 加重平均スコア 3.50 以上 │
　│  A   │ 加重平均スコア 3.00〜3.49│
　│  B   │ 加重平均スコア 2.00〜2.99│
　│  C   │ 加重平均スコア 2.00 未満 │
　└──────┴───────────────────┘

　６．KPI設定の良い例・悪い例
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　■ 良い例（行動KPI）

　　タイトル　：新規顧客獲得プロセスの標準化
　　定義　　　：新規顧客を3件獲得し、アプローチから受注までのプロセスを
　　　　　　　　言語化して部内に共有する
　　完了条件　：① 3件の新規受注があること
　　　　　　　　② アプローチシートを作成し部内共有済みであること
　　S基準　　：3件以上獲得＋共有資料を他者が再現できる水準で作成
　　A基準　　：3件獲得＋共有資料を作成
　　B基準　　：2件獲得または資料未完成
　　C基準　　：1件以下の獲得または取り組みなし

　　▷ 完了条件が明確・S/A/B/C の境界が明確・成果との接続が説明できる

　■ 悪い例

　　タイトル　：営業頑張る
　　定義　　　：しっかり営業する
　　完了条件　：（記載なし）
　　S基準　　：よくできた
　　A基準　　：普通にできた

　　▷ 評価基準が主観的・完了条件なし・S/Aの差が不明確

　７．KPI変更・差し戻しルール
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　● 変更が認められるケース
　　　① 組織変更・担当変更により業務内容が大きく変わった場合
　　　② 外部環境（大口案件の突発・天災等）で達成可否が本人の意志と無関係な場合
　　　③ 部長判断により目標設定が著しく不適切だったと認められた場合

　● 変更が認められないケース
　　　・「難しそうだから」という本人希望のみによる変更
　　　・期末直前（最終2週間以内）の変更申請

　● 変更の手順
　　　① 本人が変更申請をKPIシートの変更記録欄に記入
　　　② 部長が承認・変更日・変更理由を記録
　　　③ 変更後の基準で期末評価を実施

　８．KPI評価での注意事項
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　● 目標値の水準設定
　　・A評価（=期待通り）の目標値は「普通に頑張れば届く水準」に設定する
　　・毎期Sばかりになるような低すぎる目標は承認しない
　　・毎期Cになるような高すぎる目標も不適切

　● 外部依存KPIへの対処
　　・「市場動向次第」「取引先の都合次第」の比率が高いKPIは
　　　行動KPIと組み合わせてプロセスも評価できるよう設計する

　● 評価の一貫性
　　・同じ実績でも評価者によって判定が変わらないよう、
　　　完了条件を事前に客観的な言語で定義しておく
　　・キャリブレーション（評価者横断補正）で部署間の差を調整する
