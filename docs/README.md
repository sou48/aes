# docs

査定評価システムのWebアプリ開発リポジトリへ持ち込むための参照ドキュメント一式。

この `docs/` 配下は、現行制度の業務ルールとテンプレートを、開発側で読みやすい単位に整理した複製である。
制度変更が入った場合は、元リポジトリ側の正本を更新したうえで、必要に応じて本ディレクトリへ反映する。

---

## 構成

### `00_overview`

- `OVERVIEW.md`
  - 制度全体の概要
- `52_システム仕様書_Webアプリ化基本設計.txt`
  - Webアプリ化に必要な基本仕様

開発着手時は、まずこのディレクトリから読む。

### `10_rules`

- 評価方式
- 部署別マッピング
- 評価項目
- 部長評価

業務ルールの正本に相当するディレクトリ。
DB設計、権限制御、画面分岐、評価ロジックはここを基準に実装する。

### `20_templates`

- 統合評価シート
- フィードバック通知書
- KPIシート
- タスクログシート
- 昇給シミュレーション

画面設計、PDF設計、CSV入出力設計のベースに使う。

### `30_ops_reference`

- 管理者用運用マニュアル
- Redmineタスクログ運用ガイド

実運用の流れ、レビュー手順、連携運用の背景確認用。

### `40_master_samples`

- 制度概要の前提情報
- 会社・部署情報
- 部署役割一覧
- 従業員名簿

初期データ設計、seedデータ、テストデータ作成の参考用。

## 使い方の目安

- 画面要件を切るとき
  - `52_システム仕様書_Webアプリ化基本設計.txt`
  - `40_統合評価シート_テンプレート.txt`
  - `41_フィードバック通知書_テンプレート.txt`

- 評価方式ごとの処理を実装するとき
  - `20_評価方式_部署別マッピング.txt`
  - `21_KPI評価方式_設計.txt`
  - `22_タスクログ評価方式_設計.txt`
  - `23_行動基準評価方式_設計.txt`

- マスタと初期データを作るとき
  - `24_評価方式_部署別マッピング一覧.csv`
  - `40_master_samples/02_部署役割一覧.csv`
  - `40_master_samples/従業員名簿.csv`

- 帳票と通知書を作るとき
  - `40_統合評価シート_テンプレート.txt`
  - `41_フィードバック通知書_テンプレート.txt`

---

## 補足

- この `docs/` は開発参照用であり、制度運用の正本は元リポジトリ側にある
- `AGENTS.md` や `XX_` で始まる保留ファイルは本ディレクトリには含めていない
- 社員向け説明資料は開発実装の直接根拠ではないため除外している
