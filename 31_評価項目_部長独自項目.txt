　本書の位置づけ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　部長の評価は以下2層で構成される。

　┌──────────────────────────────┬──────────────────────────────────────┐
　│ 共通項目（全部署・全部長）         │ 部署独自項目（本書）                   │
　├──────────────────────────────┼──────────────────────────────────────┤
　│ 30_評価項目_部署別設計.txt に定義  │ 各部長の業務責任領域に固有の評価項目   │
　│ された部長評価（全部署共通）の候補 │ 四半期ごとに達成状況を評価する         │
　│ 項目群から、役員が適切な項目を選択 │                                        │
　│ して評価する                       │                                        │
　└──────────────────────────────┴──────────────────────────────────────┘

　・独自項目は各部長につき3項目（役割遂行・成果創出・組織貢献に各1項目）
　・共通項目のウェイト基本方針は
　　役割遂行 45% ／ 成果創出 30% ／ 組織貢献 25%
　・独自項目は、共通項目で拾いきれない「その部署特有の責任」を補完するために置く
　・独自項目は、共通項目と重複しすぎない内容を優先する

　【営業部】葉田
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　業務領域：
　　営業課（訪問営業・見積・大型案件の受注）と
　　受注販売課（EC受注処理・モール商品管理・出荷手配）の2課を統括する。

　■ 独自項目

　─────────────────────────────────────────────────────────────────
　【役割遂行】　営業課・受注販売課の2課マネジメント
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　訪問営業（営業課）と EC処理・モール運用（受注販売課）という
　　　性格の異なる2課を、それぞれの業務特性に合った形で機能させているか。
　　　課をまたいだ情報共有・連携の仕組みが機能しているかも評価する。

　　S │ 2課それぞれの強みが発揮されている。課間の引き継ぎ・連携がスムーズで
　　  │ 課題が即座に共有される仕組みが稼働している。
　　A │ 2課ともに計画通り機能しており、大きな連携トラブルがない。
　　B │ 一方の課に課題が集中している、または課間の情報連携が滞ることがある。
　　C │ 2課の役割・責任範囲が不明確で、機能不全または対立が発生している。

　─────────────────────────────────────────────────────────────────
　【成果創出】　受注・売上に直結する EC施策・大型案件の推進
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　モール（楽天・Yahoo等）の商品ページ改善・新規出品・キャンペーン施策など、
　　　部長として EC事業を能動的に動かしているか。
　　　また大型案件については、営業スタッフへの関与・支援が売上貢献に結びついているか。

　　S │ EC施策または大型案件への関与によって、部門売上・受注に直接貢献できる
　　  │ 成果（前期比・目標超過）が出ており、そのプロセスを言語化できる。
　　A │ EC施策・大型案件に対して部長として適切に関与し、成果につながっている。
　　B │ スタッフに任せきりで部長としての能動的な関与が薄い。
　　C │ EC施策・大型案件への関与がほぼなく、部門の機会損失が生じている。

　─────────────────────────────────────────────────────────────────
　【組織貢献】　物流事業部・クリエイティブ事業部との業務連携品質
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　受注処理→出荷指示（物流事業部）、ECサイト・商品ページ（クリエイティブ事業部）
　　　という主要な連携先との関係を良好に保ち、部門間トラブルを防いでいるか。

　　S │ 連携部署から「営業部との協働がスムーズ」という評価がある。
　　  │ 課題発生時に自部署起因かどうかに関わらず解決に動いている。
　　A │ 定期的な情報共有・確認を実施し、連携品質を維持している。
　　B │ 問題が起きてから対応する後手の対応が多い。
　　C │ 連携トラブルが繰り返し発生し、他部署からの不満が報告されている。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
　【クリエイティブ事業部】坂本
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　業務領域：
　　制作課（グループ全Webサイト・サーバー管理）と
　　開発課（Synaple：Access＋PostgreSQL の基幹システム開発・保守）の2課を統括する。

　■ 独自項目

　─────────────────────────────────────────────────────────────────
　【役割遂行】　Webサイト群・Synapleの安定稼働と品質管理
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　グループが抱える全Webサイトおよび基幹システムSynapleが安定稼働しているか。
　　　障害・不具合の発生時に適切な対応・再発防止策が講じられているかを評価する。

　　S │ 四半期中の重大障害ゼロ。発生した軽微障害には即対応済み＋再発防止策を実施。
　　  │ サイト品質・Synaple品質に対する他部署からの高評価がある。
　　A │ 障害発生時に迅速に対応し、稼働品質を概ね維持できている。
　　B │ 障害対応に遅れがある、または再発防止策が不十分で同様の問題が繰り返す。
　　C │ 重大障害が発生し業務影響が出た、または対応体制が機能していない。

　─────────────────────────────────────────────────────────────────
　【成果創出】　Synaple開発ロードマップの推進と技術的成果
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　Synapleの機能改善・新規開発がロードマップ通りに進んでいるか。
　　　また制作課・開発課の技術力向上につながる取り組みが推進されているかを評価する。

　　S │ Synaple開発が計画を上回るペースで進捗、または業務効率化への波及効果が出ている。
　　  │ 技術的な挑戦（新技術導入・設計改善）に対して具体的な成果がある。
　　A │ ロードマップ通りに開発・保守が進み、品質基準を満たしたリリースができている。
　　B │ 開発遅延が発生している、または技術的負債の解消が進んでいない。
　　C │ ロードマップが形骸化し、開発・保守が後手後手になっている。

　─────────────────────────────────────────────────────────────────
　【組織貢献】　他部署からの技術支援要求への対応・社内技術力の底上げ
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　制作課・開発課は社内唯一の技術部門として、他部署からの技術的な相談・依頼に
　　　応える責務がある。部長として受入れ方針を決め、社内技術力向上に貢献しているか。

　　S │ 他部署の技術支援要求に対して明確な受入基準を設け迅速に対応。
　　  │ 勉強会・技術共有などを通じて社内全体の技術レベル向上に貢献している。
　　A │ 他部署からの依頼に概ね対応し、良好な関係を維持できている。
　　B │ 対応可否の判断が遅い、または支援が不完全で他部署から不満が出ている。
　　C │ 他部署の依頼をほぼ断るか無視しており、社内からの信頼が低下している。

　【マーケティング企画部】桝井
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　業務領域：
　　リスティング広告（Google・Yahoo）の運用・保守、
　　メルマガの作成・送信・効果測定、ブログ記事の執筆を担当する。

　■ 独自項目

　─────────────────────────────────────────────────────────────────
　【役割遂行】　マーケティング施策の年間計画とPDCAサイクルの管理
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　広告・メルマガ・ブログという複数施策を統合的に計画し、
　　　四半期ごとにPDCAを回して次の施策に活かせているかを評価する。

　　S │ 全施策の四半期計画が明文化され、期末に効果検証と次期計画へのフィードバックが
　　  │ 完結している。施策間（広告・メルマガ・ブログ）の連携も意図的に設計されている。
　　A │ 各施策の計画・実行・振り返りが四半期内に機能している。
　　B │ 施策が個別に動いており、横断的な計画管理・効果検証が不十分。
　　C │ 施策が場当たり的で、振り返りや次期計画への反映がされていない。

　─────────────────────────────────────────────────────────────────
　【成果創出】　マーケティング投資全体のROI管理
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　広告費・工数を含めたマーケティング投資全体の費用対効果を部長として把握し、
　　　適切な予算配分・施策選択の判断をしているかを評価する。

　　S │ 投資配分の見直しにより、前期比でROIが改善している。
　　  │ 費用対効果の根拠を数値で役員に説明できる。
　　A │ マーケ予算を計画内で運用し、主要指標（ROAS・流入・開封率）を概ね達成している。
　　B │ 予算管理が担当任せで、部長として投資全体を把握・判断できていない。
　　C │ 予算超過、または主要指標が複数未達で改善策も不明確。

　─────────────────────────────────────────────────────────────────
　【組織貢献】　営業部・クリエイティブ事業部へのマーケ情報の還元
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　マーケ部門が持つ顧客行動データ・広告効果データを営業部・クリエイティブ事業部に
　　　活用可能な形で提供し、全社的な売上・制作活動に貢献しているかを評価する。

　　S │ マーケデータを元にした提案が営業・クリエイティブの施策に採用されており、
　　  │ 波及効果（受注増・サイト改善等）が確認できる。
　　A │ 定期的にマーケデータを他部署に共有し、活用を促している。
　　B │ データは持っているが他部署への共有・活用提案が少ない。
　　C │ データが部門内で完結しており、他部署に還元されていない。

　【施工事業部】石川
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　業務領域：
　　依頼を受けた工事案件の担当（ラック施工・倉庫修繕等）と
　　美原倉庫の庫内設備の修繕・管理を担う。

　■ 独自項目

　─────────────────────────────────────────────────────────────────
　【役割遂行】　工事案件の受注から完工までの工程管理
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　部長として、複数の工事案件の進捗・人員配置・顧客対応を管理し、
　　　工期遵守・品質・安全を同時に担保できているかを評価する。

　　S │ 担当全案件の工期遵守率100%。安全事故ゼロ。顧客満足の声が複数ある。
　　  │ 案件の優先度判断・人員配置が適切で、スタッフからも信頼されている。
　　A │ 全案件を工期通りに管理し、クレームなく完工できている。
　　B │ 案件管理に課題があり、工期遅延・品質問題が1件以上発生している。
　　C │ 複数案件で問題が発生し、対応が後手に回っている。

　─────────────────────────────────────────────────────────────────
　【成果創出】　美原倉庫の設備維持・改善管理
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　施工事業部が管轄する美原倉庫の庫内設備について、
　　　劣化・修繕の優先管理および改善提案を主導しているかを評価する。

　　S │ 設備の予防保全計画を策定し、予期せぬ設備停止ゼロを達成。
　　  │ コスト削減または作業効率向上につながる設備改善を1件以上実施。
　　A │ 設備の劣化・修繕箇所を把握し、優先度をつけて管理できている。
　　B │ 問題が発生してから対応する後手の管理が中心。
　　C │ 設備管理が機能しておらず、急な故障・停止が繰り返し発生している。

　─────────────────────────────────────────────────────────────────
　【組織貢献】　営業部・物流事業部との施工・倉庫連携
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　施工案件の受注情報を営業部と適切に連携し、
　　　美原倉庫設備に関する情報を物流事業部と共有して業務を円滑に進めているかを評価する。

　　S │ 案件情報・設備状況を関連部署に先回りして共有し、
　　  │ トラブルゼロ・他部署から連携品質の高い評価を受けている。
　　A │ 定期的な情報共有が機能しており、大きな連携トラブルがない。
　　B │ 情報共有が不定期で、連携の漏れ・遅れが発生することがある。
　　C │ 他部署との連携が機能しておらず、現場トラブルや不満が発生している。

　　  │ 指標の達成状況を定期的に可視化し、スタッフへ共有している。
　　A │ 主要3指標を概ね目標内で維持し、問題発生時も早期に対処できている。
　　B │ いずれかの指標で目標を下回り、根本的な改善策が不十分。
　　C │ 複数指標で慢性的な問題があり、オペレーション品質が低下している。

　【成果創出】　在庫・コスト効率の改善
　─────────────────────────────────────────────────────────────────
　　評価の視点：

　　　倉庫運営の収益貢献に部長として取り組んでいるかを評価する。

　　S │ 在庫回転率の改善またはコスト削減について、前期比で具体的な数値改善がある。
　　  │ 施策の内容・効果を役員に報告できる。
　　A │ 予算内で運営し、在庫・コスト管理を適切に実施できている。
　　B │ 現状維持に留まっており、能動的な改善取り組みが少ない。
　　C │ コスト超過・在庫過多・過少在庫が慢性化し改善策もない。

　─────────────────────────────────────────────────────────────────
　【組織貢献】　営業部・施工事業部との出荷・在庫連携の品質
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　営業部からの出荷指示への対応精度・スピード、
　　　施工事業部と共有する美原倉庫の設備・在庫情報の連携品質を評価する。

　　S │ 出荷指示に対して期待以上のスピード・精度で対応。
　　  │ 在庫状況を関連部署にプッシュ型で共有し、先回りした調整ができている。
　　A │ 定期的な情報共有・確認が機能し、連携上の大きなトラブルがない。
　　B │ 問題が発生してから連絡する後手対応が多い。
　　C │ 連携トラブルが繰り返し発生し、他部署からの信頼が低下している。

　　S │ 全部署のニーズヒアリングが完了し、優先度ロードマップが明文化・更新されている。
　　  │ 四半期内に計画したプロジェクトを全て着手・完了または進捗報告できている。
　　A │ 主要プロジェクトを計画通りに推進し、対象部署から満足の声がある。
　　B │ プロジェクト管理が属人的で、一部のプロジェクトが停滞または忘れられている。
　　C │ 効率化プロジェクトが形骸化しており、推進状況を説明できない。

　─────────────────────────────────────────────────────────────────
　【成果創出】　AIツール・システム導入による全社工数削減の実績
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　事業推進部の核心成果であるAIツール・業務システムの導入が、
　　　実際に社内の工数削減・業務改善につながっているかを数値で評価する。

　　S │ 四半期内に導入・稼働したツール・システムで、対象部署の工数削減効果が
　　  │ 定量的に実証されており（例：月○時間削減）、利活用率も70%以上を維持。
　　A │ 導入したツール・システムが対象部署で稼働し、改善効果が確認できている。
　　B │ ツールは導入したが定着していない、または効果測定ができていない。
　　C │ 導入したツールが使われていない、または開発が頓挫している。

　─────────────────────────────────────────────────────────────────
　【組織貢献】　他部署との橋渡し・社内AI活用文化の醸成
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　事業推進部が「支援部門」として各部署の信頼を得ているか。
　　　また社内にAI・DXに対する前向きな文化を広げる取り組みをしているかを評価する。

　　S │ 全部署から「事業推進部に相談したい」という声が定期的にある。
　　  │ AI活用研修・社内勉強会等を通じて社員のリテラシーが向上している実績がある。
　　A │ 依頼部署からの満足度が高く、次の相談も来ている。
　　  │ 情報発信（研修・共有）を四半期1回以上実施している。
　　B │ 相談を受けた部署のみ対応し、能動的な情報発信が少ない。
　　C │ 部署間の橋渡しが機能せず、孤立したプロジェクトになっている。

　■ 独自項目

　─────────────────────────────────────────────────────────────────
　【役割遂行】　労務・経理処理の法令遵守と正確性の統括
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　給与計算・勤怠管理・社会保険手続き・経理処理が、
　　　法令・社内規則に則って正確に期日通り完了しているかを部長として統括できているかを評価する。

　　S │ 四半期中の労務・経理処理に重大ミスゼロ。法改正への先手対応も完了。
　　  │ 処理ミスが起きない仕組み（チェック体制・マニュアル）を整備・更新している。
　　A │ 全処理が期日通り・ミスなく完了しており、法令遵守に問題がない。
　　B │ 軽微なミスが1〜2件発生している、または法改正への対応が遅れることがある。
　　C │ 重大なミス・法令違反リスクのある処理が発生している。

　─────────────────────────────────────────────────────────────────
　【成果創出】　採用活動・社内制度整備の推進
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　採用活動における計画充足と候補者の質、および就業規則・社内制度の
　　　整備・改定を四半期内に前進させているかを評価する。

　　S │ 採用計画を充足し内定承諾率が高い。かつ制度整備を1件以上完了・周知。
　　  │ 社員からの制度・ルールに関する問合せ件数が改善（減少または解決率向上）している。
　　A │ 採用活動が計画通り進捗し、制度整備または改定を1件以上着手・完了している。
　　B │ 採用が計画を下回っている、または制度整備に着手できていない。
　　C │ 採用計画が大きく未達、または制度に問題があっても放置されている。

　─────────────────────────────────────────────────────────────────
　【組織貢献】　全社員へのサポート品質と経営情報の提供
　─────────────────────────────────────────────────────────────────
　　評価の視点：
　　　総務部は全社員と役員の両方に接する部署として、
　　　社員からの相談・問合せに誠実に対応し、経営に必要な数値・情報を
　　　タイムリーに役員へ報告できているかを評価する。

　　S │ 社員からの感謝・信頼の声が複数ある。経営数値を独自分析し、
　　  │ 役員の意思決定に貢献できる提言を行った実績がある。
　　A │ 社員問合せに丁寧・正確に対応し、経営報告を期日通りに実施している。
　　B │ 対応が受け身・遅い場合がある、または経営報告に漏れ・遅延が発生している。
　　C │ 社員から不満の声が多い、または経営報告が機能していない。

　評価運用上の注意事項
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　● 独自項目の評価者は役員（社長・役員）が担う
　　　→ 部長自身が評価することはない
　　　→ 評価結果はキャリブレーション会議で確認する

　● 評価根拠の記録
　　　→ 評価者は各項目の判定根拠（具体的な事実・数値）を必ずコメントとして記録する
　　　→ 「なんとなくA」は認めない

　● 独自項目のウェイト調整
　　　→ 部署の置かれた状況（立上げフェーズ・縮小期など）に応じて、
　　　　役員判断でウェイトを増減させることができる
　　　→ 変更する場合は四半期初に対象部長に通知する

　● 独自項目の見直し
　　　→ 年次評価完了後、役員と全部長でレビューを行い、
　　　　次年度に向けて独自項目を更新する

　● 共通項目との整理
　　　→ 「新サービス・新商品・新仕組みの創出」
　　　　「メンバーの挑戦提案を引き出すマネジメント」など、
　　　　全部長共通で見るべき観点は 30_評価項目_部署別設計.txt に集約する
　　　→ 本書では、各部長の管掌領域に固有の責任と期待役割を定義する
